# Security Policy

We take security reports seriously. Please report vulnerabilities privately.

## Supported versions

We provide security fixes for the latest version on `main`. We do not backport
security fixes to older revisions unless explicitly noted.

## Reporting a vulnerability

Preferred: GitHub Security Advisories (if enabled for the repo).
Email: security@justco.build

Please include:

- A clear description of the issue and impact.
- Steps to reproduce (POC if possible).
- Affected paths or versions.
- Any relevant logs, stack traces, or screenshots.

Please do not open public issues for security reports.

## What to expect

- We aim to acknowledge reports within 2 business days.
- We will keep you updated as we investigate and remediate.
- We will request a reasonable period of confidentiality while we fix the issue.
- We can credit reporters in release notes if desired.

## Safe harbor

We welcome good-faith security research that:

- Avoids privacy violations, data destruction, or service disruption.
- Uses test accounts and minimal data.
- Reports findings promptly and privately.

Activities that are not covered by safe harbor include:

- Denial-of-service testing or spam.
- Social engineering or phishing.
- Physical attacks or threats.
